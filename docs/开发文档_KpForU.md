# KpForU 开发文档（手机端 + 手表端 + 后端）

## 1. 文档目标与范围
本文档用于说明 KpForU 新应用结构、开发环境设计、核心功能高层设计、共享数据样例，以及数据流转与展示方式。

适用对象：
- 产品经理
- 架构师
- 前后端开发与测试同学

---

## 2. 新应用结构及开发环境设计（必含项）

### 2.1 新应用整体结构
| 层级 | 子系统 | 主要职责 | 关键技术 |
|---|---|---|---|
| 客户端层 | 手机端（kpforu_phone） | 模板管理、复习卡片、语音上传、学习总结导出、个性化设置 | Flutter、Provider、HTTP |
| 客户端层 | 手表端（kpforu_watch） | 轻交互专注计时、复习确认、快速语音采集、壁纸同步 | Flutter、Riverpod、通知/振动 |
| 服务层 | API 服务（server） | 鉴权、业务编排、Leitner 复习规则、语音建卡、AI 总结与导出 | FastAPI、SQLAlchemy、Pydantic |
| 数据层 | 关系数据库 + 文件存储 | 结构化学习数据与音频文件存储 | PostgreSQL/SQLite、storage/voice |
| AI 能力层 | Qwen 兼容接口 | 语音转写、卡片草稿生成、总结文本生成 | DashScope OpenAI-Compatible API |

### 2.2 代码目录结构（简化）
| 目录 | 说明 |
|---|---|
| client/kpforu_phone/lib | 手机端业务代码（页面、状态、API 封装） |
| client/kpforu_watch/lib | 手表端业务代码（计时、复习、语音） |
| server/app/api | 后端 REST 路由层 |
| server/app/services | 业务服务层（Leitner、Qwen 客户端等） |
| server/app/models | 数据模型层 |
| server/tests | 后端测试代码 |
| docs | 项目文档 |

### 2.3 开发环境设计
| 类别 | 建议版本/配置 | 说明 |
|---|---|---|
| 操作系统 | Windows 10/11（开发机） | 当前项目已在 Windows 环境联调 |
| Flutter SDK | 3.10+ | 手机端与手表端统一技术栈 |
| Dart | 随 Flutter 版本 | 状态管理与 UI 开发 |
| Python | 3.10+ | 后端 FastAPI 运行环境 |
| 包管理 | pip / pub | 分别管理后端与前端依赖 |

## 4. 主要功能高层设计（High Level Design Document）

### 4.1 逻辑分层
| 层次 | 说明 | 输入/输出 |
|---|---|---|
| 表现层（UI） | 页面渲染与交互事件收集 | 用户点击/输入 -> 视图状态变化 |
| 应用层（Store/Provider/Riverpod） | 状态管理、请求编排、重试与轮询策略 | UI 事件 -> API 请求模型 |
| 接口层（API Client） | 鉴权头、超时控制、错误映射 | 业务请求 -> JSON 响应 |
| 服务层（FastAPI Services） | 业务规则、模型调用、数据聚合 | REST 入参 -> 业务结果 |
| 数据层（DB/Storage） | 持久化用户、卡片、会话、音频 | 结构化记录 + 文件 URL |

### 4.2 关键业务流程（摘要）
| 业务流程 | 步骤 |
|---|---|
| 设备绑定登录 | 手表注册 -> 绑定配对 -> 获取 JWT -> 双端带 Token 访问 |
| 时间流专注 | 拉取模板 -> 启动会话 -> 阶段切换 -> 会话完成上传 |
| 复习闭环 | 拉取到期卡片 -> done/snooze -> Leitner 计算 next_review_at |
| 语音建卡 | 上传音频 -> 后端异步处理 -> 轮询草稿状态 -> 用户确认入库 |
| 学习总结导出 | 聚合学习数据 -> 调用模型生成摘要 -> 生成并返回 PDF |

### 4.3 稳定性设计要点
| 主题 | 设计 |
|---|---|
| 超时控制 | 服务端模型超时 + 客户端请求超时 + 客户端轮询总超时 |
| 失败降级 | AI 失败时生成可编辑 fallback 草稿/总结 |
| 一致性 | 云端为事实源，客户端离线行为联网后重放 |
| 可观测性 | 通过 LLM_CALL、VOICE_DRAFT、LLM_SUMMARY 等日志关键词追踪 |

## 6. 数据流转及展示方式详细说明（必含项）

### 6.1 数据流主链路
| 序号 | 数据流阶段 | 数据形态 | 展示位置 |
|---|---|---|---|
| 1 | 用户在手机配置模板 | JSON（模板定义） | 手机时间流页模板列表 |
| 2 | 手表拉取模板并执行 | JSON（阶段列表 + 时长） | 手表计时页倒计时 |
| 3 | 专注结果上报后端 | JSON（session） | 手机概览页统计、手表历史页 |
| 4 | 用户上传语音文件 | multipart（二进制音频） | 手机/手表上传进度提示 |
| 5 | 后端转写并生成草稿 | JSON（transcript + card draft） | 手机语音草稿确认页 |
| 6 | 复习行为写回 | JSON（review event） | 手机复习页、手表复习页 |
| 7 | AI 汇总并导出 | 文本 + PDF 文件 | 手机导出结果入口 |

### 6.2 展示策略
| 场景 | 展示策略 |
|---|---|
| 手机端复杂信息 | 卡片分组 + 明细页（统计、草稿、导出） |
| 手表端轻交互 | 大按钮 + 最小文本 + 圆屏安全边距 |
| 异步任务反馈 | 统一状态标签：queued/processing/done/failed |
| 失败提示 | 错误码 + 可执行建议（重试/编辑/稍后） |

### 6.3 安全与合规展示规则
| 规则 | 说明 |
|---|---|
| 鉴权 | 所有业务请求携带 JWT |
| 时间 | 统一存 UTC，前端按本地时区显示 |
| 文件 | 上传大小受限，超限立即拒绝 |
| 敏感信息 | 日志不输出密钥，仅记录追踪字段 |
