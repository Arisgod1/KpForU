# **OPPO Watch 应用合作介绍**

## **基础介绍**

应用生态是智能手表的重要特点之一，OPPO Watch系列作为OPPO独立自主研发的智能手表，致力于与开发者一起探索用户使用的新场景，打造智能穿戴设备的新体验。ColorOS Watch是专为OPPO Watch定制的操作系统，基于Android系统，根据手表形态的特性，对功能、交互、界面展示进行了深度定制，为用户提供自然、贴心、轻盈的使用体验。考虑到智能手表的应用开发是一个新的领域，OPPO Watch现提供UI Kit、公共组件包、功耗标准文档等多方面材料，帮助开发者解决设计、开发过程中的相关问题，强烈建议您在设计和开发之前阅读此文档。

## **应用设计**

### **UI规范**

为了功耗考虑，区别于手机，手表的背景颜色主体为黑色，您可参考系统UI规，我们提供了包括界面布局、颜色、字体等多方面的规范，同时提供快速便捷的设计控件，帮助您更快速地实现适合手表的应用设计。

### **交互方式**

手表屏幕尺寸较小，为了用户更好的体验，我们建议您参考以下交互方式：

| 交互方式 |          | 描述                                                         |
| -------- | -------- | ------------------------------------------------------------ |
| 点击     | 单击     | 最常用交互，用于大部分操作，包括选择、启动应用，触发功能等   |
|          | 双击     | 一般用户放大图片或者地图操作                                 |
|          | 长按     | 用于低频复杂的操作，如长按编辑、删除等； 运动界面也会使用长按结束，避免用户误操作 |
|          | 拖动     | 低频操作，如用于向上拖动删除操作，相比滑动需要较高的操作难度和成本，可避免误操作，如Applist拖动进行应用排序 |
|          | 捏       | 手表屏幕小，多点触控操作困难，不建议使用                     |
|          | 拉       |                                                              |
| 滑动     | 全局右滑 | 系统全局响应右滑退出或返回上一层级，一般情况下，应用应当响应右滑返回上一界面或退出应用。 应用界面从左侧边缘右滑，滑动时当前界面跟随手势向右侧滑动； 滑动超过屏幕一半，则响应返回上一级； 未超过页面，则页面左滑返回正常状态,无上一级界面，则右滑退出该应用 |
|          | 左右滑   | 应用在非屏幕左侧边缘可进行左右滑动，用户切换应用内功能界面。 存在多个页面时，建议添加翻页符以提示用户有多个页面 |
|          | 上滑     | 表盘界面向下滑动进入控制中心，向上滑动进入通知中心。 应用页面长按屏幕边缘可以呼出控制中心或通知中心。 应用内上下滑动可查看当前屏幕范围无法查看的信息，因手表屏幕较小，采用上下滑动方式减少不同页面层级的跳转，可以更加方便用户的操作ng |
|          | 下滑     |                                                              |
| 按键     | 导航键   | 侧边上方按键，用户应用与Applist\表盘直接切换，不能用于其他功能 |
|          | 功能键   | 功能键用于开关机、快捷拉起应用、最近任务，一般情况下不用于应用内功能，功能暂不对外开放。 |

## **应用开发**

### **公共控件**

为了提高提高开发效率并和ColorOS Watch视觉规范保持统一，我们提供了公共控件包及demo文件，开发过程中可以导入控件包后直接使用。如您需要该控件，请通过客服联系我们。

### **网络连接**

请注意网络权限为用户的敏感权限，应用在使用网络前应对该权限进行声明或者申请，用户不同意之前不得进行网络连接活动。

1）蓝牙网络：手表通过蓝牙与手机进行连接，连接后通过蓝牙共享手机网络(强烈建议应用使用https网络协议）。当手机和手表连接状态下，用户最常使用的是蓝牙网络，网络速度在80KB左右。

2）WLAN：手表可独立连接WiFi进行联网，在手表端设置项-网络与连接打开wlan后添加网络即可，使用方式同手机。目前手表端网络不支持添加需要认证的网络。

3）eSIM：是一种新型的SIM卡，内嵌于设备芯片中，无需单独插卡，开通后直接使用，区分独立号码和一号双终端。手表可开通eSIM后，开启移动数据即可访问网络，使用方式同手机。但用户对于话费和流量消耗仍会关注，所以在使用移动流量的情况下，建议开发者对用户的网络状态进行必要提醒，避免额外费用的产生。eSIM开通相关问题可在“健康App-eSIM与移动网络”进行了解，或咨询当地运营商。

### **前后台机制**

用户通过点击启动应用，将应用唤到前台。通过点击导航键（或右滑）的方式把应用收到后台（或退出）。考虑到手表的屏幕较小，以及电池容量偏小的情况，当应用在前台或者后台时，您应关注以下内容：

1）在后台长时间运行的应用（如音乐、导航类应用），接入表盘顶部的indicator，提示用户应用正在后台运行。提供用户快速返回应用入口的同时，防止应用在后台长时间停留而用户无感知造成的功耗问题（如用户点开地图应用但并未进行导航，App音频播放暂停等），相关介绍见[Indicator规范及调用方法](https://open.oppomobile.com/documentation/page/info?id=11286)。

2）出于功耗考虑，OPPO Watch不支持应用在后台运行，若您的应用有重要进程需要在后台运行，可以添加前台服务，由于前台服务优先级高，不容易被杀。但请注意，前台服务会带来较高的功耗，因此请谨慎使用。同时若应用的功耗/内存等过高，仍会存在被系统杀掉进程的可能。另外，不同的Android版本中对于前台服务存在权限要求，因此为了更好地兼容不同平台，使用时建议您添加 FOREGROUND_SERVICE 权限，这是普通权限，因此系统会自动为请求权限的应用授予此权限。

3）应用在前后台运行过程中，会存在功耗问题，为了对功耗问题进行规范化处理，我们制定了相关标准及实现建议，请您务必阅读《应用功耗设计指导规范》。

### **音乐控制器接入**

音频类应用可直接适配我们的音乐控制器，方便用户统一管理正在播放的音频内容，无需单独接入indicator。同时适配后，用户可以直接在音乐适配器内快速选择需要播放的音乐app，并直接控制音乐播放，优化用户体验（如在运动过程中，无需退出运动，即可选择自己喜欢或常用的音频app进行播放）。具体适配方案，请查看《音乐控制器三方播放器适配方案》。

### **功耗规范**

智能手表的特点是内置电池容量偏小，内部结构紧凑，散热空间比较有限，佩戴贴合皮肤，对热的传导比较敏感，而功耗问题会造成续航短和温度升高的问题，所以我们对应用的功耗有较高的要求。开发者在应用开发过程中应该始终关注功耗表现，在唤醒、持锁、网络等各个场景下，应按照我们的规范和要求作出相应调整，否则会影响应用的审核及上架，具体要求及规范见《第三方应用功耗设计指导规范》，请务必阅读。如果在开发过程中有功耗相关的问题，可以联系我们进行技术咨询。

### **权限及声明**

为了让操作系统进行监管控制，相关监管机构要求移动智能终端内，第三方应用调用敏感权限时，要给用户相关提示或可以让用户确认的能力，来防范安全威胁。因此在涉及权限，尤其是敏感权限时，应用需要通过应用声明及权限弹窗明示用户，具体的规则同[Android权限要求](https://developer.android.com/guide/topics/permissions/overview?hl=zh-cn)。同时若应用涉及权限使用及用户的数据处理等内容，应当在应用启动时，有明显可见的方式供用户查看《隐私政策》及《用户协议》，且当内容变更时，应当及时地更新相关链接或文本。
